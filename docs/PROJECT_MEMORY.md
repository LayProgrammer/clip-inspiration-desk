## 1. 项目一句话

剪辑灵感台是一个面向有大量视频素材但缺少剪辑思路的创作者工具。用户上传一批素材后，系统帮助用户理解素材内容，并生成可执行的剪辑方案、字幕文案、镜头选择建议和粗剪时间线。

核心定位：

> 上传一堆素材，它帮你发现里面能剪成什么故事。

它不是一开始就做剪映或 Premiere 的替代品，而是先做“素材创意导演”。

## 1.1 产品边界校正

用户明确提醒：本项目不是剪辑软件，也不应该和剪映、美图秀秀、Premiere 等成熟工具竞争时间线编辑、特效编辑、模板商城或完整导出能力。

必须长期坚持的产品边界：

- 做“剪辑灵感提供”，不做“剪辑软件替代”。
- 做素材理解、故事发现、剪辑方向、镜头使用建议、字幕文案、风格建议、特效创意建议。
- 可以生成粗剪脚本和可执行时间线说明，但不要把核心体验做成拖拽剪辑器。
- 可以导出方案文本、分镜脚本、剪辑清单，后续可导出给剪映/Premiere 使用的辅助文件，但不把专业剪辑能力作为主战场。
- 前端可以借鉴剪映/美图秀秀的视觉质感和创作工具效率，但产品定位必须是“AI 灵感导演”，不是“轻量剪辑器”。

判断一个功能是否该做：

```text
如果它帮助用户发现素材能剪成什么、怎么组织故事、如何表达情绪，就做。
如果它主要是在复刻剪辑软件的操作能力，就暂缓或不做。
```

## 2. 用户背景

用户是零基础小白，希望 Codex 帮忙从 0 开发。用户负责表达想法、体验产品、给反馈；Codex 负责产品规划、界面设计、前端、后端、AI 工作流、视频处理和本地运行调试。

开发方式：

- 先做可运行的本地 Web App。
- 先验证核心体验，不一口气做完整商业系统。
- 边开发边看效果，逐步升级。

## 3. 产品核心问题

很多用户的问题不是“不会点剪辑按钮”，而是：

- 素材太多，不知道从哪里开始。
- 不知道这些素材能剪成什么主题。
- 不知道哪些片段适合开头、中段、结尾。
- 不知道怎么组织节奏、字幕、音乐、特效。
- 有表达欲，但缺一个“剪辑导演”帮他把素材变成故事。

本产品要解决的是剪辑前期的灵感和结构问题：

```text
素材混乱 -> 理解素材 -> 生成方向 -> 生成粗剪脚本 -> 后续自动粗剪
```

## 4. MVP 定义

第一版名称暂定：

```text
剪辑灵感台 v0.1
```

第一版必须实现：

- 新建项目。
- 上传多个视频素材。
- 展示素材列表。
- 读取基础视频信息：文件名、大小、格式、时长、分辨率。
- 生成视频缩略图。
- 用户填写素材背景和目标风格。
- AI 生成 3 个剪辑灵感方案。
- 每个方案包含主题、适合平台、建议时长、情绪风格、剪辑结构、镜头使用建议、字幕建议、音乐建议、特效建议、标题和发布文案。
- 用户选择一个方案后，生成粗剪脚本。
- 粗剪脚本以时间线形式展示每一段使用哪个素材、推荐时间段、字幕、效果和剪辑目的。
- 支持复制方案和粗剪脚本。

第一版暂不做：

- 用户登录。
- 在线支付。
- 云端部署。
- 手机原生 App。
- 完整时间线拖拽编辑器。
- 高级特效编辑器。
- 多人协作。
- 大规模视频云转码。
- 商业级版权音乐库。

## 6. 产品页面规划

### 6.1 项目列表页

用户进入应用后直接看到工作台，不做营销落地页。

功能：

- 新建项目。
- 查看历史项目。
- 展示项目名称、素材数量、最近更新时间、当前状态。

状态建议：

- 待上传。
- 待填写目标。
- 分析中。
- 已生成方案。
- 已生成粗剪脚本。
- 已导出草稿。

### 6.2 项目上传页

功能：

- 拖拽上传多个视频。
- 显示上传进度。
- 显示素材列表。
- 显示视频缩略图、时长、大小、分辨率。
- 支持删除单个素材。
- 用户填写素材背景和目标风格。

用户输入示例：

```text
这些素材是什么？
这是我和朋友去成都玩三天拍的视频，想剪成小红书风格。

你想要什么感觉？
轻松、好笑、有生活感，不要太网红。
```

### 6.3 素材分析页

第一阶段可以只展示基础信息和用户备注。

后续加入：

- 关键帧。
- 画面标签。
- 语音转写。
- 画面质量判断。
- 推荐用途。

素材标签示例：

- 人物互动。
- 街景。
- 食物特写。
- 风景空镜。
- 夜晚。
- 室内。
- 运动镜头。
- 情绪镜头。
- 可做开头。
- 可做转场。
- 可做结尾。
- 画面较抖。
- 重复素材。

### 6.4 灵感方案页

系统根据素材信息和用户描述生成 3 到 5 个方案。

每个方案结构：

```text
方案名称：
适合平台：
建议时长：
核心情绪：
一句话故事：
开头钩子：
中段结构：
结尾方式：
推荐素材：
字幕风格：
音乐建议：
特效建议：
封面建议：
发布标题：
发布简介：
为什么适合这批素材：
```

必须避免：

- 只写“可以做旅行 vlog”这种空话。
- 建议使用不存在的素材类型。
- 不引用任何具体素材或标签。

### 6.5 粗剪脚本页

用户选择一个方案后，生成可执行时间线。

时间线条目结构：

```text
00:00-00:03
素材：video_003.mp4 00:02-00:05
用途：开场钩子
字幕：第一次来这里，我就知道会记很久
效果：轻微推近，暖色调，保留一点环境音
```

用户可以通过自然语言修改：

- 再短一点。
- 更搞笑一点。
- 不要出现正脸。
- 多用食物镜头。
- 把结尾换成日落。
- 字幕更像小红书。
- 节奏更像抖音。

- 时长。

- 方案能根据用户输入变化。
- 方案包含具体素材引用。

### 阶段 5：粗剪脚本生成

目标：

- 用户选择方案后生成时间线。
- 时间线包含素材、时间段、字幕、效果、用途。
- 用户可以通过自然语言重新生成或局部调整。

完成标准：

- 用户可以照着脚本去剪。
- 时间线结构稳定，可以为后续自动导出复用。

### 阶段 6：自动粗剪导出

目标：

- 根据时间线自动剪切片段。
- 拼接视频。
- 添加字幕。
- 添加简单转场。
- 添加背景音乐或保留原声。
- 导出 MP4。

完成标准：

- 可以生成一个可下载的粗剪草稿。
- 第一版导出效果不追求精修，但必须可播放、节奏基本可用。

## 10. AI 输出原则

生成灵感方案时，输入应包含：

- 项目描述。
- 用户目标平台。
- 用户想要的情绪和风格。
- 素材列表。
- 每个素材的基础信息。
- 每个素材的标签和摘要。
- 已知限制，例如不要露脸、不要太网红、控制在 30 秒内。

生成要求：

- 输出 JSON，便于页面渲染和后续保存。
- 至少生成 3 个明显不同的方向。
- 每个方向必须引用具体素材 ID 或文件名。
- 不要建议素材里没有的画面。
- 不要只写泛泛的创意词。
- 要说明为什么这个方向适合这些素材。

粗剪时间线要求：

- 总时长符合方案建议。
- 每段有明确用途。
- 每段引用具体素材和时间段。
- 避免一个素材使用过长。
- 开头 3 秒必须有钩子。
- 结尾要有明确收束。

## 11. UI 风格方向

产品应该像创作工作台，不像营销网站。

设计关键词：

- 安静。
- 清晰。
- 年轻但不花哨。
- 有创作感。
- 信息密度适中。
- 让用户感觉素材被整理出来了。

避免：

- 大面积空洞 hero。
- 纯模板商城感。
- 过度装饰。
- 所有东西都做成卡片堆叠。
- 单一紫蓝渐变 AI 风格。

主界面应直接展示可用功能：

- 项目。
- 素材。
- 灵感方案。
- 粗剪脚本。

> 我们做的是剪辑灵感提供，不是剪辑软件。不要偏航去做剪映已经做得很强的能力。

## 15. 2026-06-07 侧边栏取舍与最新界面原则

用户反馈“侧边栏有啥用”，因此当前版本已经把左侧工具栏思路彻底取消。原因是：音频、字幕、画面调节等一级按钮容易让产品看起来像剪辑器，而且对零基础用户来说会造成“不知道点了能干嘛”的负担。

最新主界面应该坚持：

- 一级入口只保留三步任务：导入素材、描述目标、生成灵感。
- 三步任务必须有真实动作：上传素材、聚焦目标输入框、生成或重新生成灵感包。
- 素材库、导演监看台、灵感导演是工作区，不是剪辑工具箱。
- 底部结构叫“带去剪映的执行清单”，必须明确这是给用户复制到剪映/其他剪辑工具里执行的文字方案，不是在本应用里剪片。
- 声音、字幕、画面风格、标题、封面、钩子、风险提醒都应该作为“灵感包结果”展示，不应该拆成装饰性导航按钮。
- 后续新增功能时优先问：它是否帮用户更快从素材里找到故事、方向和第一版执行方案？如果只是模拟剪辑软件操作，先不要做。

## 16. 2026-06-07 流程制与封面版本

用户提出“单页面很没趣，做成流程制更好，最好再做一个酷炫封面”。当前界面已经从单页工作台升级为“封面 + 流程阶段”的体验：

- 新增封面阶段：定位为“AI 素材导演”，第一屏强调把一堆素材变成能开剪的故事，而不是替代剪映。
- 顶部流程改为可点击阶段导航：封面、导入素材、描述目标、生成灵感、执行清单。
- 主体区域按阶段显示重点工作区：导入素材阶段突出素材库和监看台；描述目标阶段突出创作目标输入；生成灵感阶段突出灵感导演；执行阶段突出可带去剪映的清单。
- 上传素材成功后自动进入“描述目标”阶段；生成灵感成功后自动进入“生成灵感”阶段。
- 流程卡从三张扩展为四张，新增“照着清单去剪映执行”，继续强调本产品输出执行方案，不在本产品里剪片。
- 移动端封面做了压缩，避免首屏过长；平板四步流程为两列，手机为一列。

后续继续升级时，流程制应保持“下一步很明确”的原则。不要恢复无意义侧边栏，也不要把封面做成纯营销页；封面必须服务于用户进入创作流程。

## 17. 2026-06-07 资深产品经理视角的下一阶段判断

用户要求继续以资深产品经理视角思考“还应该如何开发”。当前判断如下：

### 17.1 产品定位继续收窄

剪辑灵感台的核心不是“能剪”，而是“帮用户决定怎么剪”。后续所有功能必须服务于一个闭环：

> 用户上传素材 -> 系统理解素材 -> 系统给出取舍和故事判断 -> 用户拿着清单去剪映等工具里执行。

如果一个功能只是模拟剪映、Premiere、美图秀秀已经成熟的操作，例如时间线精剪、滤镜调参、贴纸模板、转场商城、音乐库、字幕编辑器，就不应该作为主线开发。

### 17.2 下一阶段最值得做的核心能力

优先级一：素材诊断

- 告诉用户这批素材“最适合剪什么”，而不是只生成几个标题。
- 给出“最佳开头素材”“必用素材”“可舍弃素材”“适合做转场的素材”“适合做结尾的素材”。
- 给出素材缺口提醒，例如缺少结尾镜头、缺少人物反应、缺少环境交代、素材重复太多。
- 给出风险提醒，例如节奏会散、主题不清、开头不够抓人、素材太少不适合长视频。

优先级二：灵感路线评估

- 每个灵感方向应该有明确评分：新手友好度、素材匹配度、发布潜力、剪辑难度。
- 不只告诉用户“可以这样剪”，还要告诉用户“为什么这条路线更值得先剪”。
- 给用户一个推荐路线，而不是把 3 个方向平铺给用户自己猜。

优先级三：执行清单升级

- 执行清单要变成“剪映旁边照着做”的清单。
- 每一步都要包含：用哪个素材、看哪一类画面、放在第几秒、字幕写什么、声音怎么处理、不要做什么。
- 支持一键复制“粗剪顺序”“字幕文案”“发布标题/封面建议”“继续追问 AI 的提示词”。

优先级四：新手引导

- 零基础用户不应该被要求会写 prompt。
- 描述目标阶段应该提供可点选的创作意图，例如：想发小红书、想做旅行日记、想做搞笑反差、不要露脸、保留现场声、想要高级感。
- 当用户什么都不写时，系统也应该能根据素材给一个保守但可执行的方案。

### 17.3 建议新增的页面/流程

在现有“封面、导入素材、描述目标、生成灵感、执行清单”基础上，后续可以把生成结果拆成更有价值的几个结果视图：

1. 素材体检报告
   - 展示这批素材的整体判断。
   - 让用户知道这堆素材有没有故事、缺什么、最该先看哪几段。

2. 路线选择页
   - 展示 3 条剪辑路线。
   - 每条路线有评分、适合平台、预计时长、剪辑难度和推荐理由。
   - 默认高亮“最推荐先剪”的路线。

3. 执行包页
   - 用户选定路线后，再展示详细剪映执行清单。
   - 这页应当是本产品最核心的交付物。

4. 复盘追问页
   - 给用户 3 到 5 个继续追问 AI 的按钮，例如“帮我更搞笑一点”“控制在 20 秒”“不要露脸重排一次”“生成小红书标题”“生成抖音开头台词”。

### 17.4 当前可以立刻开发的下一个版本

下一个版本建议命名为“素材体检报告版”。它的目标不是新增复杂编辑能力，而是让用户上传素材后立刻觉得：

> 这软件真的看懂了我这堆素材，并且知道我下一步该干什么。

建议交付：

- 后端类型新增 `MaterialDiagnosis` 或扩展 `InspirationReport`。
- 本地规则和 AI prompt 都输出：
  - `bestOpeningAssets`
  - `mustUseAssets`
  - `optionalAssets`
  - `missingShots`
  - `storyPotentialScore`
  - `editingDifficulty`
  - `recommendedRouteId`
  - `whyThisRoute`
- 前端在“生成灵感”阶段优先展示素材体检结论，再展示灵感路线。
- “执行清单”阶段根据推荐路线生成更具体的步骤。

### 17.5 产品成功标准

短期不以“能不能直接剪出视频”为成功标准，而以下列体验为标准：

- 用户上传 5 到 20 段素材后，能在 1 分钟内知道最适合剪什么。
- 用户不用懂剪辑术语，也能看懂下一步怎么做。
- 用户愿意把生成的清单复制到剪映旁边照着执行。
- 用户感觉“这不是模板网站，也不是剪辑器，而像一个帮我看片子的创意导演”。

### 17.6 明确暂缓的功能

暂缓：

- 完整时间线编辑器。
- 复杂滤镜、调色、音轨编辑。
- 模板商城。
- 直接视频导出作为主线。
- 做成通用 AI 聊天软件。

可以保留为未来增强：

- 基于 FFmpeg 的自动粗剪预览。
- 导出剪映/Premiere 可参考的结构文件。
- 更强的视觉模型分析关键帧。
- 多项目素材库和历史灵感包管理。

右上角新增“本地设置”按钮，面板包含：

- Provider 选择：local / zhipu / openai。
- 智谱 / OpenAI key 配置。
- 文本模型配置。
- 视觉模型配置。
- key 状态显示。
- 本地 storage 占用。
- generated 目录占用。
- 清理本地素材与项目按钮。

重要安全点：

- 已保存 key 不会回显到前端。
- key 输入框留空时不会覆盖已有 key。
- 保存后写入 `.env.local`，并同步更新当前进程环境变量。

  - “本地设置”按钮能打开面板。

  - 页面无应用级 console error。

已知情况：

- `next build` 会有一个 Turbopack NFT tracing warning，来源是单机设置接口需要读写项目根目录 `.env.local` 并扫描本地 storage/generated 目录。构建成功，功能不受影响。后续如果想消除该 warning，可以考虑把本地设置文件迁移到 `storage/settings.env`，但当前为了开源用户习惯，仍保留 `.env.local`。

### 19.6 下一步建议

下一步继续沿单机开源方向做：

1. 导出 Markdown 文件，而不只是复制到剪贴板。
2. 路线专属执行清单。
3. 设置向导：首次启动时引导用户配置 key。
4. 安全检查脚本：发布前扫描 `.env.local`、`storage/`、`public/generated/` 是否被纳入 git。
5. 桌面安装包预研：Electron 或 Tauri，但不要现在就重构。
