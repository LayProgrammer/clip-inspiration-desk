# 内测小白评测记录

日期：2026-06-12

## 评测身份

假设用户是刚接触剪辑的小白，手里有一批素材，但不知道如何开始，不希望在本产品里完成剪辑，只希望拿到能带去剪映、Premiere 或 CapCut 执行的灵感方案。

## 本轮结论

产品方向成立，首屏已经能表达“这不是剪辑器，而是剪辑灵感台”。真正影响小白投入使用的不是功能数量，而是每一步是否告诉用户下一步该做什么，以及生成结果是否足够可执行。

## 发现的问题

1. 首屏流程清楚，但缺少强烈的“下一步”操作提示。小白看到 5 个阶段按钮时，容易不知道应该点“生成灵感包”还是先去“导入素材”。
2. 执行清单是核心价值，但它和“灵感方向、素材体检、灵感包、追问”之间的关系还需要更明确。小白需要知道：先选路线，再生成执行包，再追问微调。
3. 旧版本内测追问记录可能出现编码异常，问题文本显示成问号，会让用户误以为当前版本坏了。
4. 追问“压缩到 20 秒”时，模型可能仍然输出 40 秒以上的时间线。对小白来说，这会直接降低信任。
5. 当前功能已经能用，但后续更适合继续优化“开剪前准备”而不是增加复杂剪辑功能。

## 本轮已修改

1. 增加“小白下一步建议”操作台。
   - 根据当前状态自动提示：上传素材、填写目标、生成灵感、查看路线、生成清单、复制执行包。
   - 按钮接入真实动作，不是静态提示。

2. 修复追问历史显示体验。
   - 如果旧记录的问题文本疑似编码异常，页面显示为“历史追问（旧版本编码异常）”。
   - 保留原回答，并给出小提示，避免用户误判软件崩坏。

3. 强化追问提示词。
   - 识别“20 秒、30 秒”等时长目标。
   - 要求模型新的执行顺序不得超过目标时长。
   - 对“不要露脸、标题封面、字幕文案”等高频追问加入硬性约束。
   - 禁止编造素材里不存在的镜头。

4. 优化本地兜底追问。
   - 没有 AI 或 AI 失败时，本地规则也会按目标秒数生成短时间线。
   - 短清单会写出明确的秒数范围。

## 下一版建议

1. 增加“AI 连通性测试”按钮，让用户知道 API key 是否真的可用。
2. 在执行清单里增加“一键改成 15/20/30 秒”的结构化按钮，而不只依赖自由追问。
3. 给每条路线增加“小白难度解释”，说明为什么这条容易或难。
4. 增加“封面标题工作区”，把标题、封面构图、前 3 秒钩子集中展示。
5. 增加示例项目或演示素材，让第一次打开的用户不用先找素材也能体验完整流程。
